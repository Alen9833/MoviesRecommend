# 电影推荐系统功能测试用例

## 1. 文档说明

- 项目名称：电影推荐系统
- 测试目标：验证系统核心业务流程是否满足毕业设计演示和论文撰写要求
- 测试类型：黑盒功能测试
- 测试环境：Windows + Python 3 + Django + SQLite 测试库
- 测试重点：用户注册、登录、搜索、评分、历史记录、推荐结果

## 2. 功能测试案例

### 用例 1：用户注册

- 用例编号：FT-001
- 功能模块：用户注册
- 测试目标：验证新用户能够使用合法数据完成注册
- 前置条件：系统已正常启动，注册用户名和邮箱未被占用
- 测试数据：
  - 用户名：`david`
  - 密码：`abc123`
  - 确认密码：`abc123`
  - 邮箱：`david@example.com`
- 测试步骤：
  1. 打开注册页面 `/movie/register`
  2. 输入用户名、密码、确认密码和邮箱
  3. 点击“注册”按钮
- 预期结果：
  1. 页面提交成功
  2. 系统跳转到首页
  3. 数据库新增一条用户记录
  4. 新用户信息可用于后续登录

### 用例 2：用户登录

- 用例编号：FT-002
- 功能模块：用户登录
- 测试目标：验证已注册用户能够成功登录并写入会话信息
- 前置条件：系统中已存在用户 `alice / 123456`
- 测试数据：
  - 用户名：`alice`
  - 密码：`123456`
  - 记住登录：选中
- 测试步骤：
  1. 打开登录页面 `/movie/login`
  2. 输入用户名和密码
  3. 提交登录请求
- 预期结果：
  1. 登录成功后跳转首页
  2. 会话中保存 `user_id`
  3. 系统能够识别当前登录用户

### 用例 3：电影搜索

- 用例编号：FT-003
- 功能模块：电影搜索
- 测试目标：验证系统能够根据关键字模糊查询电影
- 前置条件：数据库中存在电影 `Action Hero`、`Drama Life`
- 测试数据：
  - 搜索关键字：`Action`
- 测试步骤：
  1. 访问搜索接口 `/movie/search?keyword=Action`
  2. 查看返回结果列表
- 预期结果：
  1. 页面返回状态码为 200
  2. 结果中包含 `Action Hero`
  3. 结果中不包含 `Drama Life`
  4. 页面上下文中保留用户输入的关键字，便于界面回显

### 用例 4：电影评分修改

- 用例编号：FT-004
- 功能模块：电影详情与评分
- 测试目标：验证登录用户可以在详情页提交评分，并对已有评分进行更新
- 前置条件：
  - 用户 `alice` 已登录
  - 用户之前已对 `Action Hero` 评分 `4.5`
- 测试数据：
  - 电影：`Action Hero`
  - 新评分：`5`
  - 新评论：`updated`
- 测试步骤：
  1. 打开电影详情页 `/movie/detail/<movie_id>`
  2. 在评分区域输入新分数和评论
  3. 提交评分表单
- 预期结果：
  1. 提交后跳转回当前电影详情页
  2. 数据库中该用户对该电影的评分记录被更新，而不是重复新增
  3. 新评分值为 `5`
  4. 新评论内容为 `updated`

### 用例 5：评分历史与偏好分析

- 用例编号：FT-005
- 功能模块：评分历史
- 测试目标：验证系统能够展示用户评分记录，并生成类型偏好分析数据
- 前置条件：
  - 用户 `alice` 已登录
  - 用户对 `Action Hero` 和 `Drama Life` 均有评分记录
- 测试数据：
  - `Action Hero` 评分：`4.5`
  - `Drama Life` 评分：`3.0`
- 测试步骤：
  1. 打开评分历史页 `/movie/history/<user_id>`
  2. 查看历史评分列表
  3. 查看雷达图数据是否生成
- 预期结果：
  1. 页面成功加载
  2. 页面展示两条评分记录
  3. 雷达图指标中包含 `Action`、`Drama`
  4. 雷达图数值包含 `4.5` 和 `3.0`
  5. 系统能够反映用户对不同电影类型的偏好强度

### 用例 6：删除评分记录

- 用例编号：FT-006
- 功能模块：评分历史管理
- 测试目标：验证用户可以删除自己的评分记录
- 前置条件：
  - 用户 `alice` 已登录
  - 用户已对 `Drama Life` 产生评分记录
- 测试数据：
  - 待删除电影：`Drama Life`
- 测试步骤：
  1. 在评分历史页点击删除按钮
  2. 系统调用 `/movie/del_rec/<movie_id>`
  3. 页面跳转回评分历史页
- 预期结果：
  1. 删除请求执行成功
  2. 数据库中对应评分记录被删除
  3. 页面跳转到当前用户的评分历史页
  4. 后续历史列表中不再出现该记录

### 用例 7：个性化推荐

- 用例编号：FT-007
- 功能模块：电影推荐
- 测试目标：验证系统能够基于相似用户的历史评分生成未看电影推荐
- 前置条件：
  - 用户 `alice` 已登录
  - 系统中存在多个用户评分数据
  - `alice` 已看过 `Action Hero`、`Drama Life`
  - 相似用户看过 `Space Mission`、`Mixed Epic`
- 测试数据：
  - 当前用户：`alice`
  - 推荐候选电影：`Space Mission`、`Mixed Epic`
- 测试步骤：
  1. 打开推荐页面 `/movie/recommend`
  2. 查看返回的推荐电影列表
- 预期结果：
  1. 页面正常返回推荐结果
  2. 推荐列表包含 `Space Mission` 和 `Mixed Epic`
  3. 推荐列表不应包含用户已看过的 `Action Hero`、`Drama Life`
  4. 说明推荐算法具备基础协同过滤能力
